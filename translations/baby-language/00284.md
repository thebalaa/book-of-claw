## The Translation

The timestamp prefix setting got simpler! Instead of complicated options, you can now just say:
- `true` — yes, add timestamps (uses your system's timezone)
- A timezone string like "America/New_York" — add timestamps in that timezone
- `false` — no timestamps

Most people just want `true`, so that's now the default. Keep it simple!

## The Changes

Simplifies the `timestampPrefix` configuration to accept either a boolean (true/false) or a timezone string. Defaults to true for convenience, automatically using the system timezone.
