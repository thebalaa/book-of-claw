## The Translation

Oops! A URL like "https://example.com/new/page" was triggering the /new directive! That's not right.

This fix avoids matching directives when they appear inside URLs. Tests verify this works correctly.

## The Changes

Fixes directive detection to avoid false matches inside URLs, with test coverage.
